# Glossary — Ubiquitous Language

The shared vocabulary for this project: one canonical term per concept, each
defined in a single line. The goal is that the team, the PRDs, the code, and
Claude all use the *same word* for the *same thing*. Keep it minimal — capture
domain concepts, not generic software terms.

`/grill-me` reads this file before an interview (to ground its questions in the
project's established terms) and proposes new entries at the end of one. You own
the file — edit, prune, and rename freely.

## Format

One bullet per term, kept alphabetical:

- **Term** — one-sentence definition. _(aka: synonym to avoid; not: easily-confused term)_

The `aka:` / `not:` note is optional — add it only when there's a competing word
worth ruling out. That disambiguation is what makes the language *ubiquitous*.
